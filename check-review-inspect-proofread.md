# 英语辨析：Check / Review / Inspect / Proofread

在日常学习、工作和开发中，我们经常遇到表示「检查、审查」的词汇。虽然它们的中文意思有些接近，但其适用的场景、正式程度以及侧重点有很大不同。

---

## 快速对比表

| 词汇 | 中文核心释义 | 侧重点 | 场景特征 | 常见搭配 |
| :--- | :--- | :--- | :--- | :--- |
| **Check** | 检查 / 核对 / 确认 | 确认**对错、是否存在、是否工作**。最常用、最宽泛。 | 快速、日常、结果通常是「是/否」。 | `check the email`, `check the list` |
| **Review** | 评审 / 审查 / 回顾 | 评估**质量、效率或可行性**，给出反馈或进行调整。 | 系统性、专业性、常伴随意见和修改。 | `code review`, `book review` |
| **Inspect** | 检验 / 视察 / 稽查 | 检查是否**符合法律、安全规范或质量标准**。 | 官方、正式、通常由专家或第三方机构执行。 | `inspect the factory`, `inspect goods` |
| **Proofread** | 校对 / 校稿 | 寻找并纠正**拼写、语法、标点和排版**等字面错误。 | 文本/代码注释发布前的最后一步（关注低级错误）。 | `proofread the article`, `proofread a draft` |

---

## 详细辨析

### 1. Check（最常用：确认对错与状态）

**Check** 是最普遍、最日常的词。它的核心目的是快速确认「某事是否正常」、「是否正确」或「是否存在」。

- **特点**：通常耗时较短，得出的结论往往是二选一的（例如：对/错、有/无、开/关）。
- **常见场景**：
  - 确认门锁了没有（`check the lock`）
  - 确认有没有拼写错误（`check for typos`）
  - 检查邮箱（`check the inbox`）
- **例句**：
  - *Please check if the server is running.* （请检查服务器是否在运行。)
  - *Double-check your code before committing.* （提交代码前再核对一遍。)

### 2. Review（系统性：评审、反馈与评估）

**Review** 强调「重新看一遍以进行评估」。它不仅是看对错，更侧重于对整体质量、结构、设计进行**分析、打分或给出改进建议**。

- **特点**：更具建设性，通常需要专业知识，且往往会产生反馈意见（feedback）。
- **常见场景**：
  - 代码评审（`code review`）
  - 绩效考核/回顾（`performance review`）
  - 书评/影评（`book/movie review`）
- **例句**：
  - *Can you review my pull request?* （你能帮我评审一下我的 PR 吗？)
  - *We need to review our marketing plan.* （我们需要评估一下我们的营销计划。)

### 3. Inspect（官方/正式：合规性与安全性检查）

**Inspect** 是非常正式的词。它通常指由专家、质检员或官方机构进行**极其仔细、严密的检查**，以确认是否符合特定的标准、法律、规范或安全要求。

- **特点**：严谨、官方、带有合规性（compliance）色彩。
- **常见场景**：
  - 视察工厂或工地安全（`inspect the factory`）
  - 海关检查物品（`inspect the baggage`）
  - 房屋质检（`home inspection`）
- **例句**：
  - *The fire department will inspect the building tomorrow.* （消防部门明天将检查这栋大楼。)
  - *The customs officer inspected our luggage.* （海关官员检查了我们的行李。)

### 4. Proofread（专注字面：拼写、语法与排版校对）

**Proofread** 的范围非常具体，专门用于**文字出版或文档发布的最后阶段**。它的目的不是去改写内容大纲，而是找出**拼写错误（typos）、语法毛病、标点符号遗漏或格式排版问题**。

- **特点**：只关注表面的文本错误（surface errors），不涉及深层的逻辑修改或重构。
- **常见场景**：
  - 文章发表前的校对（`proofread the essay`）
  - 发送重要邮件前的检查（`proofread the email`）
- **例句**：
  - *Could you help me proofread this article?* （你能帮我校对一下这篇文章吗？)
  - *I always proofread my reports twice before submitting.* （我在提交报告前总是会仔细校对两遍。)

---

## 终极记忆口诀

> 写完一篇文章，先 **Proofread** 挑挑错别字；再找朋友 **Review** 提提修改意见；发表前自己 **Check** 一下排版是否对齐；如果是给政府或大客户交差，可能还得经得起专家 **Inspect**。
