# SSH Ubuntu 24.04 远程开发


## 开发模式的利与弊

### 优势（利）
*   **原生 Linux 系统环境（核心优势）**：
    有些服务深度依赖 Linux 特有的系统调用（如 `/proc` 目录监测、`/dev/shm` 共享内存路径、POSIX 实时线程优先级调度等），远程 Ubuntu 24.04 能够完美提供这些运行条件。
*   **极佳的 IDE 远程开发体验**：
    现代 IDE（如 VS Code Remote - SSH 插件、JetBrains Gateway 等）提供无缝连接。代码编辑、代码跳转和断点调试均在 Mac 上操作，而实际编译和执行透明地发生在远程 Ubuntu 上。
*   **节省本地宿主机资源**：
    免去了在 Mac 上运行笨重虚拟机的系统开销，避免发热和电池损耗，同时能够利用高性能远程服务器加速项目编译。

### 劣势（弊）
*   **强依赖网络状况**：
    网络延迟高或抖动时，终端输入会出现滞后感，连接断开会影响开发节奏。
*   **硬件物理隔离**：
    虽然软件业务逻辑可完全仿真，但诸如 PCIe 通信、FPGA LVDS 硬件驱动、CAN 总线等与硬件设备强关联的接口，在远程服务器上依然需要使用 Mock 方式调试。
*   **多用户环境资源抢占**：
    若多名开发人员在同一台 Ubuntu 机器上共用同一套代码或运行环境，Boost IPC 共享内存和端口等全局资源可能会产生冲突。


## 注意事项与避坑指南

### 1. 共享内存残留清理
当测试或调试系统服务时，若程序异常退出或崩溃，共享内存段可能会残留在系统中，导致下一次运行失败。
*   **解决方法**：每次重新启动服务程序前，执行一次清理命令：
    ```bash
    rm -f /dev/shm/boost_ipc_*
    ```

### 2. 实时线程优先级权限（RT Priority）
一些服务中设置了 `SCHED_FIFO` 或 `SCHED_RR` 实时调度策略以保证算法高频执行的低时延。
*   **注意点**：Linux 默认不允许非 root 用户配置实时优先级，导致设置失败。
*   **解决方法**：
    *   **临时**：使用 `sudo` 运行编译出的程序。
    *   **永久**：编辑远程 Ubuntu 的 `/etc/security/limits.conf` 文件，加入以下内容允许您的用户（例如 `alex`）设置实时优先级：
        ```text
        alex   -   rtprio   99
        alex   -   memlock  unlimited
        ```

### 3. 避免多实例资源互锁
*   **注意点**：同一台 Ubuntu 机器上，同一时刻只允许运行一组相同的服务进程。前一个会话残留的进程如果不彻底杀死，会抢占共享内存槽位导致新实例启动崩溃。
*   **排查命令**：
    ```bash
    ps aux | grep -E "sys-service|phasearrayservice|process-monitor"
    ```

### 4. 使用后台终端维持工具（Tmux / Screen）
*   **注意点**：通过 SSH 直接运行程序时，一旦 Wi-Fi 抖动断开，SSH 会话关闭，运行中的编译任务或仿真进程就会被系统自动杀死。
*   **解决方法**：SSH 连上远程服务器后，优先执行 `tmux` 命令在一个会话窗口中进行编译和调试。即使网络断开，重新 SSH 连上后运行 `tmux a` 即可快速恢复现场。
