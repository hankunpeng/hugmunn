# Git force-with-lease 详解

`--force-with-lease` 是 Git 中一种**更安全的“强推” (force push) 方式**。

---

## 1. 传统的强推：`git push --force` (危险 ❌)

当你使用 `--force` 时，你是在告诉远程仓库：“**不管三七二十一，用我本地的代码直接覆盖远程的代码**。”

* **潜在风险**：如果在你最后一次拉取（`git fetch` 或 `git pull`）代码之后，**有其他同事往这个分支推送了他们的新提交**，普通的 `--force` 会毫不留情地把同事辛辛苦苦写的新代码**全部抹掉**。

---

## 2. 安全的强推：`git push --force-with-lease` (推荐/安全 

`--force-with-lease`（字面意思是“带租约的强推”）在推送前增加了一个**核对机制**：

* 它会检查：**“我本地记录的远程分支状态，与远程仓库当前的实际状态，是一致的吗？”**
* **情况 A（允许推送）**：如果在你上次 fetch 之后，**没有任何人**往这个分支推送过新代码，Git 就会认为你只修改了自己的东西，允许你强推覆盖。
* **情况 B（拒绝推送）**：如果此时**有其他人**已经推送了新提交，远程仓库的实际状态和你本地记录的不同，Git 就会**拒绝你的推送**并报错。这样你就不会意外抹掉别人的工作。

---

## 3. 形象的比喻 🏠

* **`--force`**：直接破门而入，把房间里所有的东西扔出去换成你的，不管里面是不是已经住进了别的人。
* **`--force-with-lease`**：在推门前先核对一下：“门锁和房间里的摆设还是我上次走时的样子吗？”如果发现变了（说明有人住进去了），就立刻停手，避免产生破坏。

---

## 总结

它既能满足你修改提交历史后进行强推的需求（例如在本地 `git commit --amend` 或 `git rebase` 之后），又能**确保不会在无意中覆盖掉其他人的代码**。在多人协作的项目中，应该养成用 `--force-with-lease` 代替 `--force` 的好习惯。
