# GitHub Flavored Markdown 简明教程

GitHub Flavored Markdown (GFM) 是编写 GitHub 存储库文档（如 `README.md`）、问题反馈以及合并请求的默认格式，它在标准 Markdown 语法之上增加了一系列强大且实用的扩展功能。

## 文本与格式

使用星号、底线或波浪号可以创造不同风格的强调效果。以下是常见的内联样式：

*   **加粗**：使用两个星号 `**` 或两个底线 `__` 包围需要强调的重点内容（如 `**重点**` 或 `__重点__`）。
*   *斜体*：使用单个星号 `*` 或单个底线 `_` 包围文本（如 `*斜体*` 或 `_斜体_`），适合专有名词或特殊标记。
*   ~~删除线~~：使用两个波浪号 (`~~`) 修饰作废或已删除的文字（如 `~~删除~~`）。
*   <u>下划线</u>：由于 Markdown 语法本身没有原生的下划线标记，在 GFM 中可以通过使用 HTML 的 `<u>` 标签来实现（如 `<u>下划线</u>`）。

## 标题等级

清晰的标题层级能帮助读者快速理解文档的逻辑脉络。

在文本前添加一至六个井号 `#` 即可创建对应级别的标题。官方建议在一份文档中仅使用一个一级标题作为页面标题，主干内容从二级标题 `##` 开始。

```markdown
## 这是二级标题

二级标题下的描述内容。

### 这是三级标题

三级标题下的具体细节。
```

## 列表与任务追踪

列表是整理无序或有序信息的好帮手。GFM 进一步提供了任务列表功能，用于追踪待办事项的完成情况。

如果内容没有特定的顺序，请使用无序列表；如果包含固定步骤，请使用有序列表。

*   无序列表使用连字符 `-` 或星号 `*`。
*   有序列表使用数字加句点 `1.`。
*   任务列表在无序列表符号后加上方括号 `[ ]`，完成状态写为 `[x]`。

例如：

```markdown
- [x] 撰写项目设计文档
- [ ] 编写核心业务逻辑代码
- [ ] 添加单元测试用例
```

## 引用与强调提示

引用区块常常用于摘录其他文档的内容，但在 GFM 中，它还能产生更强烈的视觉警告效果。

在段落首行使用大于号 `>` 可以创建普通引用块。GFM 专属的警告提示 Alerts 也是基于引用块创建的，非常适合在操作指南中提供关键注意事项。

> 这是普通的引用文本。
> 你可以用它来引用他人的回复或文档内容。

支持的 Alert 提示框共有 5 种，语法如下：

```markdown
> [!NOTE]
> 这是一条需要读者了解的背景信息或补充说明。

> [!TIP]
> 这里是一些有助于提高效率的建议或最佳实践。

> [!IMPORTANT]
> 关键性内容，读者必须注意以避免出错。

> [!WARNING]
> 警告信息，操作可能会带来一些非预期后果。

> [!CAUTION]
> 危险警告，可能会导致严重后果或资产损失。
```

## 代码展示

- 内联代码用单反引号包围单词，例如表示文件名 `index.js`。
- 代码块使用三个反引号包围多行代码，并在第一行结尾处写明使用的编程语言（如 `python`）。
	```python
	print("Hello, World!")
    print("Hello, GFM Code Block!")
	```

## 表格数据

在技术文档中，表格是对比多项数据和展示 API 参数的核心工具。

使用竖线（`|`）来划分不同的列，并在标题行和内容行之间加上由连字符（`-`）组成
的分隔符。冒号（`:`）还可以用于控制列内文本的对齐方式。

```markdown
| 参数名称 | 类型     | 描述           |
| :------- | :------: | -------------: |
| 左对齐   |   居中   |         右对齐 |
| `name`   | `string` |   用户的名字   |
```
| 参数名称 | 类型     | 描述           |
| :------- | :------: | -------------: |
| 左对齐   |   居中   |         右对齐 |
| `name`   | `string` |   用户的名字   |

## 数学公式支持

科学计算和工程项目常常需要在文档中严谨地表达复杂的数学公式。

你可以借助美元符号来渲染受支持的 MathJax 数学表达式。使用单个符号表示内联公
式，使用双符号表示独立成行的公式块。

内联公式演示：方程式为 `$E = mc^2$` $E = mc^2$。

复杂的公式段落：

$$
\left( \sum_{k=1}^n a_k b_k \right)^2 \leq \left( \sum_{k=1}^n a_k^2 \right) \left( \sum_{k=1}^n b_k^2 \right)
$$

## 独有特性

在 GitHub 的问题追踪器（Issues）和合并请求（Pull Requests）中直接写作时，GFM 可以与整个代码库生态产生联动。

你可以自动关联到平台上的任何相关记录、代码提交和协作者：

*   **成员提到**：使用 `@` 可以呼叫特定用户或团队（如 `@username`）。
*   **引用 Issue / PR**：使用井号 `#` 加上数字（例如 `#123`）会自动转换为对应编号的 Issue 或 PR 链接。
*   **引用 Commit**：输入代码提交的 SHA-1 哈希值前 7 位或更多位字符（例如 `8ef44aa`），GFM 会自动将其转换为对应 Commit 的超链接。
*   **自动转链接**：直接粘贴 URL 地址，GFM 会自动附加可点击的超链接。

## 扩展功能

### Emoji 表情符号

GFM 支持通过以冒号包围的表情简称来快捷输入丰富的表情符号：

- 输入 `:sparkles:` 会渲染为 ✨
- 输入 `:tada:` 会渲染为 🎉
- 输入 `:octocat:` 会在 GitHub 网页端渲染为章鱼猫吉祥物图标（属于 GitHub 专属表情）

### 脚注（Footnotes）

脚注非常适合用来为技术文档添加名词解释、来源出处或补充说明，而不会打乱正文排版：

```markdown
这是一个带有脚注的句子[^1]。

[^1]: 这是脚注的具体内容，通常会被渲染在页面底部。
```
